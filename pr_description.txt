💡 What: Added a 300ms debounce to the search input in the Admin Panel catalog.
🎯 Why: The search input was previously triggering a full re-render of the catalog (`renderAdminCatalog()`) and performing multiple `i18n` translations on every keystroke. This causes unnecessary layout thrashing and performance degradation, particularly on slower devices or when the product list grows.
📊 Impact: Significantly reduces the number of re-renders and computations during user input, leading to a much smoother and snappier typing experience. Reduces re-renders from O(keystrokes) to O(1) per typing pause.
🔬 Measurement: Open the Admin Panel, navigate to the catalog, and quickly type "торт" in the search box. The catalog list should only update once you pause typing, rather than attempting to update with each letter typed.
